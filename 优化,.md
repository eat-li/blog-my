# 对当前项目进行优化

## 1. 配置文件

1.1. 文件/server/.env的JWT_SECRET 当前为占位符默认值 ，需要修改为强随机字符串

1.2  文件server/services/uploadService.js(line 44)的getSignature()返回了accessKeyId给客户端，泄露了 OSS Access Key ID  改为服务端直传模式

1.3 从日志中移除明文密码   文件: server/seed.js (line 18)console.log('管理员账号已创建 (admin admin123)') 打印了明文密码   改为不打印密码，或建议通过环境变量设初始密码

## 2. 服务端安全

2.1 安装并配置 Helmet 安全头,使用pnpm install helmet

2.2 配置 CORS 来源限制，文件server/app.js当前允许所有来源，改为只允许前端开发服务器和生产域名（当前暂无）

2.3 添加登录接口限流，后台登录现在是无限流保护，可以暴力破解express-rate-limit 对/api/auth/login 添加限流（如：每分钟 5次尝试

2.4 安装并使用 express-validator 校验输入，如用户名、密码、邮箱，url格式，使用pnpm install express-validator
