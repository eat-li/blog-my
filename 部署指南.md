# 博客部署到阿里云 ECS (Ubuntu) 指南

## 一、服务器环境安装

### 1.1 安装 Node.js 18+

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
source ~/.bashrc
nvm install 18
nvm use 18
node -v   # 确认版本 >= 18
```

### 1.2 安装 pnpm 和 PM2

```bash
npm install -g pnpm pm2
```

### 1.3 安装 MySQL 8.0

```bash
sudo apt update
sudo apt install mysql-server -y

# 安全配置（设置 root 密码，移除匿名用户等）
sudo mysql_secure_installation

# 创建数据库
sudo mysql -u root -p
```

在 MySQL 中执行：

```sql
CREATE DATABASE `blog-saki` DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'blog'@'localhost' IDENTIFIED BY '你的数据库密码';
GRANT ALL PRIVILEGES ON `blog-saki`.* TO 'blog'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

### 1.4 安装 Nginx

```bash
sudo apt install nginx -y
sudo systemctl enable nginx
sudo systemctl start nginx
```

## 二、部署项目

### 2.1 克隆项目

```bash
# 先配置 SSH key 或在克隆时输入账号密码
sudo mkdir -p /var/www/blog
sudo chown $USER:$USER /var/www/blog
git clone <你的仓库地址> /var/www/blog
cd /var/www/blog
```

### 2.2 配置环境变量

```bash
cp server/.env server/.env.backup
vim server/.env
```

修改为生产环境配置：

```ini
# 数据库配置（改为服务器上的 MySQL）
DB_HOST=127.0.0.1
DB_PORT=3306
DB_NAME=blog-saki
DB_USER=blog
DB_PASSWORD=你的数据库密码

# JWT（务必改成你自己的强随机字符串！）
JWT_SECRET=生成一个随机的64位十六进制字符串

# 阿里云 OSS（保持不变）
OSS_REGION=oss-cn-chengdu
OSS_ACCESS_KEY_ID=你的AccessKey
OSS_ACCESS_KEY_SECRET=你的Secret
OSS_BUCKET=blog-saki
OSS_BASE_URL=https://blog-saki.oss-cn-chengdu.aliyuncs.com

# 服务端口
PORT=3000
NODE_ENV=production
```

### 2.3 安装依赖并构建

```bash
# 后端
cd /var/www/blog/server
pnpm install
node seed.js    # 创建管理员 admin / admin123

# 前端
cd /var/www/blog/client
pnpm install
pnpm build      # 产出 dist/ 目录
```

### 2.4 配置 Nginx

```bash
# 使用项目自带的 Nginx 配置
sudo cp /var/www/blog/nginx-blog.conf /etc/nginx/sites-available/blog
sudo ln -s /etc/nginx/sites-available/blog /etc/nginx/sites-enabled/

# 移除默认站点
sudo rm -f /etc/nginx/sites-enabled/default

# 检查配置
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx
```

### 2.5 启动后端服务（PM2）

```bash
cd /var/www/blog/server

# 创建日志目录
mkdir -p /var/www/blog/logs

# 启动
pm2 start ecosystem.config.js

# 设置开机自启
pm2 save
pm2 startup
# 执行上面命令输出的 sudo 命令
```

### 2.6 验证

```bash
# 检查 PM2 状态
pm2 status

# 检查 Nginx 状态
sudo systemctl status nginx

# 测试 API
curl http://localhost:3000/api/posts

# 在浏览器访问 http://你的服务器IP
```

## 三、阿里云安全组配置

登录阿里云控制台 → ECS → 安全组 → 添加规则：

| 方向 | 端口 | 协议 | 授权对象 | 说明 |
|------|------|------|----------|------|
| 入方向 | 80 | TCP | 0.0.0.0/0 | HTTP 访问 |
| 入方向 | 443 | TCP | 0.0.0.0/0 | HTTPS（配置 SSL 后开启） |
| 入方向 | 22 | TCP | 0.0.0.0/0 | SSH 远程登录 |

> 建议 22 端口限制为你本机的 IP 地址以提高安全性。

## 四、后续更新部署

每次更新代码后，在服务器上执行：

```bash
cd /var/www/blog && bash deploy.sh
```

或者手动步骤：

```bash
cd /var/www/blog
git pull
cd client && pnpm install && pnpm build
cd ../server && pnpm install && pm2 restart blog-server
```

## 五、常见问题

**Q: 页面刷新后 404？**
A: Nginx 配置中 `try_files $uri $uri/ /index.html` 没有生效，检查 nginx-blog.conf 是否正确加载。

**Q: API 请求 502？**
A: PM2 没有启动或端口 3000 没在监听。运行 `pm2 status` 检查。

**Q: 数据库连接失败？**
A: 确认 MySQL 正在运行 `sudo systemctl status mysql`，检查 .env 中的数据库密码是否正确。

**Q: 上传图片失败？**
A: 检查 OSS 配置的 AccessKey 是否正确，OSS Bucket 是否存在且权限正确。
